Der wichtigste Workflow

Dein erstes Briefing

Das machst du am häufigsten. Die Regel: je konkreter dein Input, desto besser der Output. Claude erfindet keine Zahlen oder Claims - gib ihm die echten.

Was Claude von dir braucht

Hast du nicht alles? Egal - Claude fragt nach, was fehlt.

A

Marke & Produkt

Name, was es ist, Preis, was es besser macht als andere.

B

Ziel & Zielgruppe

Was die Anzeige bewirken soll (Trial, Kauf, Leads) und für wen - so konkret wie möglich.

C

Format & Plattform

Static, Carousel, Motion, UGC-Video? Meta, Google, TikTok? Wenn unklar, schlägt Claude was vor.

D

Fakten & Angebot

Echte Claims, Studien, Bewertungen, das Offer. 1:1, nichts erfunden.

Wenn was hakt

Häufige Fragen

In meiner Claude-App gibt es kein /plugin!

Brauchst du auch nicht. /plugin gibt es nur in Claude Code (Entwickler-Tool). In der normalen App/Cowork ist die Suite org-weit bereitgestellt und damit einfach da - du musst sie höchstens einmal einschalten (siehe Suite aktivieren). Manuell hochladen ist nur ein Fallback.

Beim Upload passiert nichts / „Skills" fehlt im Menü

Dann ist die Skills-Funktion noch aus. Geh in die Einstellungen → Capabilities und aktiviere „Code execution and file creation". Bei Team/Enterprise muss das der Org-Owner in den Org-Settings freigeben.

Welche Datei lade ich hoch - die große Suite-ZIP?

Nein. Für den App-Upload nimmst du die einzelnen Skill-ZIPs (z.B. admkrs-cs-creative-briefing.zip) - eine ZIP = ein Skill. Die große Suite-ZIP ist nur für Claude Code / zum Archivieren.

Muss ich Befehle exakt tippen?

In der App gar nicht - du beschreibst deine Aufgabe normal auf Deutsch, der Skill springt automatisch an. Nur in Claude Code sollten die zwei /plugin-Zeilen genau stimmen.

Bekomme ich Updates automatisch?

Org-weit bereitgestellt (der Normalfall): ja - der Suite-Admin aktualisiert zentral, du musst nichts tun. In Claude Code: ja (Plugin-Installation aktualisiert sich). Nur beim manuellen ZIP-Upload (Fallback): bei größeren Updates die neue Skill-ZIP nochmal hochladen. Der Suite-Admin sagt im Team Bescheid, wenn's eine neue Version gibt.

Sind hier echte Kundendaten drin?

Nein. Alle Beispiele nutzen die fiktive Marke „NOVA". Kundenspezifische Karten (z.B. Verifier-Do's&Don'ts) liegen in der internen Ablage und werden zur Laufzeit mitgegeben, nicht im Repo. Die Suite enthält unsere Methodik, keine Kundendaten.
